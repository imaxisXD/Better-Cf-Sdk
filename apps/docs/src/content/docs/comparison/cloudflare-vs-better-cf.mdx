Compare these implementation paths for stateful Worker development:

- native Cloudflare path (Wrangler + manual queue / Durable Object wiring)
- primary better-cf path (`better-cf/durable-object`)
- legacy queue-only path (`better-cf/queue`)

## What You Will Achieve

- understand where the primary `better-cf/durable-object` surface improves everyday workflow speed
- identify where native Cloudflare remains the better fit
- choose a practical default per team/project stage

## Before You Start

- know your team’s queue workload profile (feature velocity vs low-level control)
- review current boundaries in [Limitations](/limitations)
- know that `better-cf/durable-object` is the current primary Alpha package in the suite

## Step 3: Compare Code Shape

### Manual Cloudflare pattern (common)

<Code
  lang="ts"
  code={`import { z } from 'zod';

type Env = { QUEUE_SIGNUP: Queue };

const signupSchema = z.object({
  email: z.string().email(),
  userId: z.string()
});

export default {
  async fetch() {
    return new Response('ok');
  },
  async queue(batch: MessageBatch<unknown>, env: Env) {
    for (const message of batch.messages) {
      const parsed = signupSchema.safeParse(message.body);
      if (!parsed.success) {
        message.retry();
        continue;
      }
      try {
        await handleSignup(env, parsed.data);
        message.ack();
      } catch {
        message.retry();
      }
    }
  }
};`}
/>

### Primary durable-object pattern

<Code
  lang="ts"
  code={`import { createSDK } from 'better-cf/durable-object';
import { z } from 'zod';

const sdk = createSDK();
export const defineWorker = sdk.defineWorker;

export const room = sdk.defineDurableObject({
  name: 'Room',
  key: z.string()
});

export const emailQueue = sdk.defineQueue({
  args: z.object({
    roomId: z.string(),
    to: z.string().email(),
    body: z.string()
  })
});

export const sendMessage = room.fn({
  args: z.object({
    body: z.string(),
    author: z.string()
  }),
  handler: async ({ storage }, message) => {
    const messages = ((await storage.get('messages')) as unknown[] | undefined) ?? [];
    messages.push(message);
    await storage.put('messages', messages);
    return { ok: true };
  }
});

export const emailQueueConsumer = emailQueue.message({
  handler: async (ctx, job) => {
    await ctx.api.room.sendMessage(job.roomId, {
      body: job.body,
      author: 'system'
    });
  }
});

export default defineWorker({
  async fetch(_request, ctx) {
    await ctx.api.emailQueue.send({
      roomId: 'general',
      to: 'team@example.com',
      body: 'hello'
    });

    return new Response('ok');
  }
});`}
/>

Expected output:

- reduced worker boilerplate and centralized queue + Durable Object behavior definition

## Troubleshooting

### Comparison looks too abstract for your team

Map each row to one current pain point (drift, setup complexity, testing cost) and run one pilot Durable Object + queue workflow with the primary surface.

### Team needs unsupported runtime behavior

Use native Cloudflare APIs directly for that path while keeping SDK for supported queue workflows.

### Decision blocked by Alpha status

Adopt the Durable Object SDK on bounded workflows first and keep fallback escape hatches explicit in architecture docs.
